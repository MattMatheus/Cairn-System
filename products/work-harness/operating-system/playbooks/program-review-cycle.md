# Program Review Cycle Playbook

Run after PM refinement or before release checkpoint.

This is a supporting/reference playbook. Use `products/work-harness/HUMANS.md` first for normal stage flow.

## Steps
1. Validate queue and artifact consistency:
   - `products/work-harness/tools/test_program_state_consistency.sh`
2. Update the active backlog readmes and current planning/control artifacts that reflect real queue order and current priorities.
3. Confirm PM TODO `Now` has actionable next items.
4. Confirm phase and ADR traceability fields for intake stories.
5. Decide whether a release bundle should be created or updated.
