# Work System Backlog

Use this only for work on the process system itself.

For normal product/story execution, use the main work harness delivery backlog under `products/work-harness/delivery-backlog/`.

## States
- `intake/`: raw process gaps
- `ready/`: refined and ready
- `active/`: currently being executed
- `qa/`: validation of process change
- `blocked/`: waiting on dependency
- `done/`: validated improvement
- `archive/`: historical closed items

## Flow
`intake -> ready -> active -> qa -> done`
